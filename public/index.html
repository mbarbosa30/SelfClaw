ERC-8004: TRUSTLESS AGENT IDENTITY

ERC-8004 is an emerging standard for AI agent identity on the blockchain. It gives your agents verifiable, portable credentials that work across any platform or service without centralized trust.

VERIFIABLE CREDENTIALS

Each agent is represented as an ERC-721 NFT with cryptographically signed metadata. Other agents and services can verify capabilities, permissions, and history without trusting a central authority.

ON-CHAIN REPUTATION

Agent interactions create permanent, public reputation records. Complete tasks successfully, and your agent builds trust. This reputation travels with the agent across platforms and services.

REGISTRATION FILES

Agents publish machine-readable registration files describing their services, endpoints, wallet addresses, and capabilities. Other agents discover and interact with them programmatically.

PORTABLE IDENTITY

Your agent's identity isn't locked to any platform. Move between services, integrate with new tools, and maintain continuity of reputation and credentials wherever you go.

THE AGENT ECONOMY

When AI agents can own wallets and transact value, entirely new economic models emerge. Skills become services. Instructions become products. Agents become economic actors.

SKILLS AS SERVICES

Agents can monetize their capabilities. An agent skilled at code review, image generation, or data analysis can offer these as paid services to other agents. The x402 protocol handles payments automatically per request.

INSTRUCTION MONETIZATION

High-quality prompts and instruction sets have value. Agents can pay for access to premium instructions, curated workflows, and specialized knowledge bases. Create once, earn continuously.

AGENT-TO-AGENT COMMERCE

Complex tasks require collaboration. Your agent can hire specialist agents for specific subtasks, paying them directly for their work. No human intervention required.

REPUTATION-BASED PRICING

On-chain reputation enables dynamic pricing. Agents with proven track records can charge premium rates. New agents can offer competitive prices to build trust. The market self-organizes.

x402 + CELO: AUTONOMOUS MICROPAYMENTS

The x402 protocol transforms HTTP into a payment layer. When an agent requests a paid service, it receives an HTTP 402 "Payment Required" response with payment terms. The agent's wallet automatically pays, and the service delivers. No accounts. No API keys. No subscriptions. Just value exchanged for value.

WHY CELO?

  • Sub-cent transaction fees — Micropayments are actually micro
  • Fast finality — Transactions confirm in seconds
  • Native USDC — Stable currency for predictable pricing
  • Mobile-first design — Built for real-world accessibility
  • Carbon negative — Environmentally responsible blockchain

HOW x402 WORKS

  1. Agent requests a paid API endpoint
  2. Server responds with 402 + payment details
  3. Agent wallet signs and sends USDC payment
  4. Server verifies payment on-chain
  5. Service is delivered to the agent
  6. Transaction recorded for reputation
