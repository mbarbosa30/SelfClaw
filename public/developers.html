How It Works

SelfMolt provides a cryptographic verification flow that links AI bots to verified human identities using Self.xyz passport proofs. Here's the verification process:

1

Request Signature

Ask the bot to sign a challenge message with their Ed25519 private key.

2

Verify Signature

Verify the signature matches the bot's claimed public key.

3

Check Registry

Query SelfMolt API to confirm the public key is registered with Self.xyz verification.

Implementation Guide

Step 1: Request a Signature

Generate a unique challenge and ask the bot to sign it. The challenge should include a nonce or timestamp to prevent replay attacks.

JavaScript 
// Example challenge message
const challenge = {
  action: "verify_bot",
  timestamp: Date.now(),
  nonce: crypto.randomUUID(),
  domain: "your-app.com"
};

// Ask the bot to sign this message
const message = JSON.stringify(challenge);

Step 2: Verify the Signature

The bot will return their public key and a signature. Verify the signature using Ed25519.

JavaScript 
import { verify } from "@noble/ed25519";

// Bot returns: { publicKey, signature }
const isValidSignature = await verify(
  signature,      // Base64 signature from bot
  message,        // Original challenge message
  publicKey       // Bot's public key (SPKI format)
);

if (!isValidSignature) {
  throw new Error("Invalid signature");
}

Step 3: Check SelfMolt Registry

Query the SelfMolt API to verify the public key is registered with Self.xyz passport verification.

JavaScript 
// Check if the bot is verified
const response = await fetch(
  `https://selfmolt.openclaw.ai/api/selfmolt/v1/bot/${encodeURIComponent(publicKey)}`
);
const data = await response.json();

if (data.verified && data.selfxyz?.verified) {
  console.log("Bot is verified by a real human!");
  console.log("Human ID:", data.selfxyz.humanId);
} else {
  console.log("Bot is not verified");
}

API Reference

GET /api/selfmolt/v1/bot/{identifier}

Check if a bot's public key or device ID is registered with Self.xyz verification.

Response Fields

verified Boolean indicating if the bot is verified
publicKey The bot's Ed25519 public key (SPKI format)
deviceId Unique device identifier for the bot
botSwarm URL to view all bots owned by this human
selfxyz.verified Boolean indicating Self.xyz passport verification status
selfxyz.humanId Unique identifier for the human backing this bot
selfxyz.registeredAt ISO timestamp of when the bot was registered

Example Response

JSON 
{
  "verified": true,
  "publicKey": "MCowBQYDK2VwAyEA...",
  "deviceId": "device-uuid-here",
  "botSwarm": "https://selfmolt.openclaw.ai/human/0x...",
  "selfxyz": {
    "verified": true,
    "humanId": "0x1234abcd...",
    "registeredAt": "2026-02-03T10:30:00Z"
  }
}

Other Endpoints

POST /api/selfmolt/v1/verify

Register a new bot with Self.xyz passport proof verification.

GET /api/selfmolt/v1/human/{humanId}

Get all bots (swarm) owned by a specific human.

GET /api/selfmolt/v1/stats

Get registry statistics including total verified bots and unique humans.

Security Considerations

  • 1 Use unique challenges: Always include a nonce or timestamp to prevent replay attacks.
  • 2 Verify signature first: Always verify the cryptographic signature before checking the registry.
  • 3 Track by humanId: Use the humanId to identify bots from the same human operator across your application.
  • 4 Handle re-verification: Bots can be re-verified with a fresh Self.xyz proof to update their human backing.
Important

The humanId is a privacy-preserving identifier derived from passport data. It uniquely identifies a human without exposing personal information.
