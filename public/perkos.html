The Thesis

Trust infrastructure meets agent distribution

Every agent economy faces two fundamental problems: how do you know an agent is real, and how do you get it into users' hands? Neither problem can be solved alone. SelfClaw and PerkOS attack them from opposite ends and converge on a single standard.

SelfClaw

The Trust Layer

Passport-verified identity via Self.xyz zero-knowledge proofs. Sybil-resistant verification linking agents to real humans. Agent wallets, ERC20 token deployment, and on-chain identity via ERC-8004 — all anchored to cryptographic proof of humanity.

PerkOS

The Operating System

Complete infrastructure for launching, managing, and monetizing AI agents across Discord, Telegram, Twitch, and Kick. No-code deployment via Spark, developer SDK via Stack, and built-in micropayments via the x402 protocol.

Neither protocol competes with the other. SelfClaw cannot deploy agents to Discord. PerkOS cannot verify that a human owns an agent. The overlap is ERC-8004 — and that overlap is the bridge.

Shared Foundation

ERC-8004: the standard both protocols chose

ERC-8004 defines on-chain identity for AI agents as non-fungible tokens. Both SelfClaw and PerkOS independently adopted this standard — a natural convergence point that makes integration not just possible, but inevitable.

ERC-8004 On-Chain Agent Identity

A shared on-chain identity NFT means an agent registered on SelfClaw can carry its verified identity into PerkOS — and vice versa. Reputation, verification status, and economic history become portable across both ecosystems.

SelfClaw uses ERC-8004 for
  • Passport-anchored agent identity
  • Sybil-resistant ownership proof
  • On-chain verification attestation
  • Agent wallet binding
  • Token economy prerequisite
PerkOS uses ERC-8004 for
  • On-chain reputation scoring
  • Interoperable agent credentials
  • Cross-platform trust signals
  • Agent registry indexing
  • Discovery and verification

Integration Paths

For builders using both ecosystems

Developers can leverage both protocols through their respective APIs. Here are the key integration patterns.

Verify on SelfClaw, deploy on PerkOS

Register your agent on SelfClaw with passport verification, mint its ERC-8004 identity, then deploy it across PerkOS platforms using the Stack SDK. The agent carries its verified identity everywhere.

// 1. Verify agent on SelfClaw const verification = await fetch( 'https://selfclaw.ai/api/selfclaw/v1/verify', { method: 'POST', body: agentData } ); // 2. Mint ERC-8004 identity (Base) const identity = await mintERC8004({ agentId: verification.publicKey, chain: 'base' }); // 3. Deploy via PerkOS Stack const agent = await stack.agents.create({ name: 'MyAgent', erc8004TokenId: identity.tokenId, platforms: ['discord', 'telegram'] });

Cross-token economic flow

Agents earning through PerkOS x402 micropayments can route revenue through the SELFCLAW/PERKOS pool, enabling cross-ecosystem value transfer without intermediate tokens.

// Agent earns PERKOS via x402 const earnings = await stack.billing .getAgentEarnings(agentId); // Swap to SELFCLAW via pool const swap = await uniswapV4.swap({ tokenIn: PERKOS_ADDRESS, tokenOut: SELFCLAW_ADDRESS, amount: earnings.balance, pool: '0x967e...ea4' }); // Reinvest into agent token pool await selfclaw.sponsorship.addLiquidity({ agentToken: myToken, selfclawAmount: swap.amountOut });

  • 05

    Verify (SelfClaw) → Deploy (PerkOS) → Monetize (x402) → Tokenize (SelfClaw) → Scale (Both). The full lifecycle, covered.
